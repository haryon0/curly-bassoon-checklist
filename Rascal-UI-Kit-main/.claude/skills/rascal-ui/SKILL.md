# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# ★★★ DESIGN PHILOSOPHY IS PARAMOUNT ★★★

# ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

> **Before any token, class, font, or component decision — internalise the philosophy.**
> Tokens encode it. Components implement it. But the philosophy is what *justifies* every choice. Without it, the rest is cargo-cult Tailwind.

## North-star: *Calm by Default*

> Quiet, low-friction, low-cognitive-load. The interface fades into the background; the user's work does not.

Rascal apps are **internal tools used daily by busy operators** — leaders scanning a dashboard at 7am, HR closing a review at end-of-quarter, finance dropping a number before a deadline. The UI's job is to **get out of the way**, not to demand attention.

Loud states (urgency, error, success) exist — but they are **exceptions**, not the baseline.

## The Four Principles (memorise — these decide every disagreement)

1. **Earn the pixel.** If an element can be removed without losing meaning, remove it.
2. **Signal beats decoration.** Color / weight / size encode meaning, not vibes. Tone choices map to semantic states.
3. **Function before form.** When UX and aesthetic conflict, function wins.
4. **Loud only when necessary.** Quiet baseline. Urgency, error, success states earn their loudness *because the rest is calm*.

If a design decision contradicts these principles, **the principles win** — not the designer's taste, not the AI's training, not "what looks nice today". Full doc: `$KIT/docs/PRINCIPLES.md` — read it once a quarter even if you think you remember it.

## The visual DNA — `okr.rascalrepublic.com`

When in doubt about what "Calm by Default + Rascal brand" *looks* like, the canonical reference is the production OKR Tracker app at **`https://okr.rascalrepublic.com`**:

- Light, warm off-white surface (no cool blue-gray).
- Brand logo (two-peak mountain mark) centered on the boot loader, on a faint blue-purple gradient.
- Thin (not chunky) blue progress ring orbiting the logo.
- "Loading OKR Tracker" in semibold near-black; subtitle in muted warm gray.
- No marketing copy. No decoration. Brand presence is the logo + the ring — nothing else.

This is the **DNA** every new Rascal app must inherit. The kit's `LoadingSpinner` template was extracted directly from this screen — use it verbatim. The static SVG preview lives at `$KIT/assets/animations/canonical-loading-preview.svg`.

## Universal guardrails (non-negotiable)

These rules apply to **every** Rascal surface — code written by hand, generated by AI (v0, Lovable, Cursor), or copied from elsewhere. No exceptions without a documented `// DESIGN: diverges from kit — reason: <why>` comment.

1. **Stone is the only neutral.** Never `gray-*` / `slate-*` / `zinc-*` in new code.
2. **Brand navy `#011132` is sidebar + auth only.** In-app CTAs use `bg-stone-900`.
3. **Emerald is the only "yes, commit" color.** Submit / Approve / Confirm = `bg-emerald-700`.
4. **Font stack starts with Aptos.** Aptos → Inter → Segoe UI → system. No deviation.
5. **Every loading state uses the branded `LoadingSpinner`.** App boot, auth boundary, suspense fallback, full-screen blocker — **mandatory**. No lone `animate-spin` circles, no skeleton-only loaders for auth/boot. The logo must be present.
6. **Emoji only from the 9-glyph allowlist.** New emoji = bug, not feature.
7. **Material Symbols Rounded for all non-emoji icons.** No Lucide, no Feather, no Font Awesome.
8. **Numbers in columns use `tabular-nums`.** Always.
9. **Calm baseline.** If something earns its loudness (urgency, error, success), the rest of the page is quiet to make room.

Treat these as hard checks. If AI-generated code violates any of them, fix the code before it merges.

> **READ THIS BEFORE ANYTHING ELSE if you are building auth, app-boot, or branded UI.**
>
> This is the single most-violated section across new Rascal apps. AI design tools (v0, Lovable, Cursor) almost always get the brand logo wrong on the first pass — they use a generic placeholder, the wrong property, or a lone spinner with no logo. Humans skim past the "three brands" detail because it's easy to miss.
>
> **The user sees the LoadingSpinner and Login screen BEFORE anything else in the app. If they are wrong, the whole product reads as off-brand from the first second.** Fix these first.

## ▸ Three brands, three palettes — corporate vs boutique villa

The Rascal Republic group spans two design temperaments:

| Brand              | Temperament                  | Body neutral                       | LoadingSpinner ring                            |
| ------------------ | ---------------------------- | ---------------------------------- | ---------------------------------------------- |
| **Rascal Republic** | Corporate, functional, calm.  | Cool stone (`bg-stone-50`)         | Blue (`border-t-blue-600` on `border-blue-200`) |
| **Rinjani Bay**     | Boutique villa, jungle + bay. | Warm cream (`#FAF7F2`)             | Turquoise (`border-t-[#0E8A8A]` on `border-[#9FD4D4]`) |
| **Samara Lombok**   | Boutique villa, sunset + sea. | Warm cream (`#FCFAF5`)             | Sunset gold (`border-t-[#D4A648]` on `border-[#EBD9A8]`) |

Universal across all three: layout grid, type scale, status semantics (emerald=success, amber=warn, red=error, blue=info), Material Symbols Rounded, emoji allowlist, voice & tone baseline, branded LoadingSpinner template. **Full per-brand palette tokens are in `$KIT/docs/BRAND_COLORS.md` — read it before building any villa surface.**

> **The ring color and the body neutral are the most-confused tokens.** Villa apps with a corporate blue spinner immediately read as "wrong brand" — even before the user notices anything else. Match the property.

## ▸ Three brands, three logo sets — never mix

The Rascal Republic group runs three properties. Every Rascal app belongs to **exactly one** of them — pick the right folder before importing any asset:

| Property            | Kit folder                              | Belongs here                                                                                       |
| ------------------- | --------------------------------------- | -------------------------------------------------------------------------------------------------- |
| **Rascal Republic** | `assets/logo/rascalrepublic/`           | OKR Tracker, EPR, Leadership Portal, Account Portal — every internal Rascal group ops tool          |
| **Rinjani Bay**     | `assets/logo/rinjanibay/`               | Any app or surface for the Rinjani Bay property (hotel ops, guest, booking, internal tools)         |
| **Samara Lombok**   | `assets/logo/samaralombok/`             | Any app or surface for the Samara Lombok property (hotel ops, guest, booking, internal tools)       |

Each set has **four files** — the variant matches the background it sits on:

| File                       | Background it sits on                | Where it goes                                                                |
| -------------------------- | ------------------------------------ | ---------------------------------------------------------------------------- |
| `<brand>-navy.svg`         | Light (white / stone-50 / gradient)  | **DEFAULT in-app + the LoadingSpinner logo** ← use this 90% of the time      |
| `<brand>-white.svg`        | Dark navy / brand gradient           | **Sidebar, login screen, dark-mode headers** ← only place white logo lives   |
| `<brand>-black.svg`        | Print, mono, fax                     | PDFs, printed reports, anywhere color is unavailable                          |
| `<brand>.png`              | Email clients (SVG often stripped)   | Email header / signature                                                      |

> **Hard rule:** importing the wrong brand's logo or the wrong variant (e.g. navy logo on navy background) is a brand violation. Always confirm the app's property before the import statement is written.

## ▸ Pattern A — First-load LoadingSpinner (the very first frame the user sees)

This screen renders **before** the user is authenticated, while the app boots, while session is being validated, and at every suspense/route boundary. **It is mandatory across every Rascal app.**

### Requirements (all must hold)

1. **Use the branded `LoadingSpinner` template** — `$KIT/templates/components/LoadingSpinner.tsx`. Never a lone `animate-spin` div. Never a plain "Loading…" text. Never a skeleton alone.
2. **Logo variant: `-navy.svg`** — the spinner sits on a light gradient (`from-blue-50 via-white to-purple-50`), so the navy logo gives contrast.
3. **Logo property matches the app's brand** — Samara Lombok app → `samaralombok-navy.svg`. Rinjani Bay app → `rinjanibay-navy.svg`. Rascal Republic ops tool → `rascalrepublic-navy.svg`. **Not always Rascal Republic.**
4. **`appName` prop matches the user-visible product name** — "Samara Booking", "Rinjani Concierge", "OKR Tracker" — not the repo name.

### Implementation — per-brand examples

**Samara Lombok app:**

```tsx
// src/components/AppBootLoader.tsx
import LoadingSpinner from '@rascal-ui-kit/templates/components/LoadingSpinner';
import logoUrl from '@rascal-ui-kit/assets/logo/samaralombok/samaralombok-navy.svg';

export default function AppBootLoader() {
  return (
    <LoadingSpinner
      appName="Samara Booking"
      logoSrc={logoUrl}
      message="Please wait while we sign you in…"
    />
  );
}
```

**Rinjani Bay app:**

```tsx
import logoUrl from '@rascal-ui-kit/assets/logo/rinjanibay/rinjanibay-navy.svg';

<LoadingSpinner appName="Rinjani Concierge" logoSrc={logoUrl} />
```

**Rascal Republic ops app (OKR Tracker, EPR, etc.):**

```tsx
import logoUrl from '@rascal-ui-kit/assets/logo/rascalrepublic/rascalrepublic-navy.svg';

<LoadingSpinner appName="OKR Tracker" logoSrc={logoUrl} />
```

### Where to wire it

- **App root** — wrap the `<App />` in a suspense / loading boundary that renders `<AppBootLoader />` until `isAuthReady === true`.
- **Auth callback page** — `<AppBootLoader message="Signing you in…" />` while exchanging the magic-link token.
- **Route suspense fallback** — `<Suspense fallback={<AppBootLoader />}>` for any lazy-loaded route.
- **Full-screen blocker** — long-running export, bulk import, anything that locks the UI.

## ▸ Pattern B — Login / auth screen (the only place brand navy goes full-bleed)

The login screen is the **single in-app surface** where brand navy `#011132` covers the whole background. It also gets the `-white.svg` variant because the logo sits on dark navy.

### Requirements (all must hold)

1. **Background:** brand gradient — `linear-gradient(135deg, #011132 0%, #022050 100%)`.
2. **Logo variant: `-white.svg`** of the app's brand.
3. **Login card:** white `bg-white rounded-2xl shadow-2xl` floating on the gradient.
4. **Submit button:** the same brand gradient — this is the ONE place the gradient button appears. After login, in-app CTAs revert to `bg-stone-900` / `bg-emerald-700`.
5. **Inputs:** `rounded-lg` + `focus:ring-2 focus:ring-[#011132]` (slightly heavier brand treatment than the in-app default `rounded-md` + stone focus ring).
6. **Right after a successful submit, transition to `<AppBootLoader />`** (Pattern A) with the *same* brand's `-navy.svg`.

## ▸ Pattern C — Programmatic brand picker (multi-tenant apps)

If one deployed app serves multiple properties (e.g. an Account Portal that lets a user log into any of Rascal / Rinjani / Samara from one screen, or a Concierge tool that switches brand by subdomain), centralise the mapping so a single state value decides everything:

```tsx
// src/lib/brand.ts
import rascalNavy   from '@rascal-ui-kit/assets/logo/rascalrepublic/rascalrepublic-navy.svg';
import rascalWhite  from '@rascal-ui-kit/assets/logo/rascalrepublic/rascalrepublic-white.svg';
import rinjaniNavy  from '@rascal-ui-kit/assets/logo/rinjanibay/rinjanibay-navy.svg';
import rinjaniWhite from '@rascal-ui-kit/assets/logo/rinjanibay/rinjanibay-white.svg';
import samaraNavy   from '@rascal-ui-kit/assets/logo/samaralombok/samaralombok-navy.svg';
import samaraWhite  from '@rascal-ui-kit/assets/logo/samaralombok/samaralombok-white.svg';

export type Brand = 'rascalrepublic' | 'rinjanibay' | 'samaralombok';

export const brandLogo: Record<Brand, { navy: string; white: string }> = {
  rascalrepublic: { navy: rascalNavy,  white: rascalWhite  },
  rinjanibay:     { navy: rinjaniNavy, white: rinjaniWhite },
  samaralombok:   { navy: samaraNavy,  white: samaraWhite },
};

export const brandLabel: Record<Brand, string> = {
  rascalrepublic: 'Rascal Republic',
  rinjanibay:     'Rinjani Bay',
  samaralombok:   'Samara Lombok',
};

// Resolve brand from subdomain / env / user selection.
export function detectBrand(): Brand {
  const host = window.location.hostname;
  if (host.includes('rinjani')) return 'rinjanibay';
  if (host.includes('samara'))  return 'samaralombok';
  return 'rascalrepublic';
}
```

Then at the call sites:

```tsx
const brand = detectBrand();

// Boot loader
<LoadingSpinner appName={brandLabel[brand]} logoSrc={brandLogo[brand].navy} />

// Login screen
<img src={brandLogo[brand].white} alt={brandLabel[brand]} className="h-10" />
```

## ▸ Sanity check — auth + boot UI (run before merging any login PR)

- [ ] The LoadingSpinner template is used at the auth boundary — **no lone spinner, no plain "Loading…" text, no skeleton-only boot.**
- [ ] The boot LoadingSpinner uses the **`-navy.svg`** of the correct brand for this app.
- [ ] The login screen uses the **`-white.svg`** of the correct brand (logo sits on dark navy).
- [ ] The app's brand actually matches the property — Samara app doesn't accidentally ship Rascal logo, Rinjani app doesn't show Samara logo.
- [ ] After login submit, the next render is `<AppBootLoader />`, not a flash of unstyled content or a blank screen.
- [ ] Login submit button uses **brand gradient `#011132 → #022050`** — this is the only in-app place that gradient appears.
- [ ] In-app CTAs (post-login dashboard buttons) are `bg-stone-900` / `bg-emerald-700`, **not** the brand gradient (gradient is reserved for the auth screen).
- [ ] Login inputs use `rounded-lg` + `focus:ring-2 focus:ring-[#011132]`; in-app inputs use `rounded-md` + stone focus ring.
- [ ] Multi-tenant apps centralise brand picking in one file (`src/lib/brand.ts`) — no scattered logo imports per page.

> **If any check fails: fix it before the PR merges.** This is the user's first impression of the product. Drift here costs disproportionately more than drift on an internal dashboard.

## Step 2 — Cheatsheet (no read required)

The most-used tokens, surfaced for fast recall. Always verify against the docs above before committing to code.

### Font stack (verbatim — set once in `index.css`)

```css
font-family:
  'Aptos', 'Aptos Display',
  'Inter', 'Inter Variable',
  'Segoe UI Variable', 'Segoe UI',
  system-ui, -apple-system, BlinkMacSystemFont,
  'Helvetica Neue', Helvetica, Arial,
  sans-serif;
```

Plus the icon-font exclusion so Material Symbols don't render as `?`:

```css
*:not(.material-symbols-rounded):not(.material-symbols-outlined):not(.material-symbols-sharp) {
  font-family: /* same stack as above */;
}
```

Self-host Inter via `import '@fontsource-variable/inter';`. Don't try to host Aptos — it resolves locally on Windows 11 / M365 and falls through to Inter elsewhere.

## Step 5 — Concrete workflow for a *new* Rascal app

When the user says "starting a new Rascal app" / "build a dashboard for X" / similar:

1. **Set up the font** — paste the font stack + icon-font exclusion into the new app's `src/index.css`. Install `@fontsource-variable/inter`.
2. **Set up the Tailwind theme** — define `rr-primary` / `rr-accent` / `rr-success` / `rr-warning` / `rr-error` via `@theme` (Tailwind 4) or `extend.colors` (Tailwind 3) so kit classes resolve. Both `rr-*` and `brand-*` aliases are acceptable.
3. **Copy the logo** — pick the brand the app belongs to; copy `*-navy.svg` for in-app, `*-white.svg` for sidebar/auth, `.png` for email.
4. **Build the shell** — sidebar `bg-[#011132]` text-white; main `bg-stone-50`; page wrapper `max-w-{4|5|6|7}xl mx-auto p-6 space-y-5`.
5. **Build the greeting** — use the greeting recipe verbatim (eyebrow + H1 + body line).
6. **Drop in patterns, don't invent** — for each section in your design, match to a pattern in `DASHBOARD_PATTERNS.md` first. Only invent when no pattern fits, and propose it back to the kit as a PR.
7. **Copy** — every user-facing string runs through the rules in `VOICE_AND_TONE.md` before merge. Imperative verbs in buttons. Time deltas in plain English (`Cycle locks in 3d`, not `2026-05-15`).
8. **Loading — MANDATORY branded spinner.** Every loading state (app boot, auth boundary, suspense fallback, full-screen blocker, route transition) uses the branded `LoadingSpinner` template with `logoSrc` set to the app's brand logo. No lone `animate-spin` rings, no plain "Loading…" text, no spinner without the logo. The logo presence is part of the brand identity — if loading is visible, the logo must be visible.
9. **Email templates** — switch font to `'Segoe UI', system-ui, -apple-system, sans-serif` (email clients strip web fonts). Same color tokens. Use `.png` logo, not SVG.

## Step 6 — Sanity checks before declaring "done"

Run these mental checks against every PR:

- [ ] Body text uses `text-stone-*` only (no `text-slate-*` / `text-gray-*`).
- [ ] Primary CTA is `bg-stone-900`, not brand navy and not teal.
- [ ] Submit / "yes, commit" actions are `bg-emerald-700`.
- [ ] Brand navy `#011132` only appears on the sidebar background and auth gradient.
- [ ] Numbers in any column use `tabular-nums`.
- [ ] Every emoji is on the allowlist (9 glyphs total) — others are Material Symbols.
- [ ] Stat displays use `text-3xl font-semibold tabular-nums` (color = tone).
- [ ] No card-per-row borders where a hairline `border-stone-100` would do.
- [ ] Smart-gate applied wherever the section *might* show misleading partial data.
- [ ] Animations are `duration-150`–`duration-300`; nothing animates layout properties.
- [ ] **Every** loading state uses branded `LoadingSpinner` with the right brand logo — no exceptions (boot, auth, suspense, route change, full-screen blocker).

If any check fails: fix the app, not the kit. Drift without a documented reason is a bug.

## Step 7 — When to extend vs. when to follow

You may **propose changes back to the kit** via PR if you hit a real gap (new pattern with 2+ real callers, missing brand variant, etc). Until that PR lands, follow the existing token / pattern even if it's not perfect. Drift across apps is much worse than imperfect tokens.

You may **NOT** silently diverge — if the app needs a deviation, log it in the file with a comment `// DESIGN: diverges from kit — reason: <why>` so the next reviewer can audit.

## Step 8 — Anti-patterns to call out immediately

Stop and warn the user whenever you see (in their code or your own draft):

1. `bg-[#011132]` on an in-app button — **fix:** use `bg-stone-900`.
2. New emoji not on the allowlist — **fix:** Material Symbol Rounded glyph.
3. `gray-*` / `slate-*` / `zinc-*` neutrals mixed with `stone-*` — **fix:** migrate to stone.
4. Submit / Approve button in teal or stone — **fix:** emerald-600/700.
5. Same metric showing different numbers in dashboard vs. email — **fix:** extract shared formatter; see `DASHBOARD_PATTERNS.md → Email-dashboard sync`.
6. Loading state without the brand logo — **fix:** use the branded `LoadingSpinner` template.
7. Card with both drop-shadow and border — **fix:** pick one (hairline border preferred).
8. Marketing copy ("powerful", "delightful", "Oops!") — **fix:** rewrite per `VOICE_AND_TONE.md`.

## Step 9 — Vercel deployment + AI design tools (v0, Lovable, Cursor)

When the user builds a Rascal app on **Vercel** and/or generates UI with an **AI design tool** (v0.dev, Lovable, Cursor compose, Bolt), the kit still applies — guardrails are non-negotiable. The challenge: AI tools default to generic shadcn/ui aesthetic and won't follow the kit unless you tell them to, in language they understand.

### 9a. Making the kit reachable from a Vercel build

Vercel build server cannot read the local OneDrive path where the kit lives. Pick one before deploying:

| Option                 | When to pick                                       | Setup                                                                    |
| ---------------------- | -------------------------------------------------- | ------------------------------------------------------------------------ |
| **Git submodule**      | Single app, kit changes rarely                     | `git submodule add <kit-repo-url> src/rascal-ui-kit`                     |
| **Monorepo workspace** | Multiple Rascal apps in one repo                   | pnpm/npm workspaces; kit becomes an internal package                     |
| **Copy into `src/`**   | Prototype, one-shot, don't care about future sync  | Copy `templates/` + `assets/logo/` into the app                          |
| **Custom registry**    | 3+ apps consuming kit; want `shadcn add` workflow  | Deploy kit as static site on Vercel + `registry.json` (see 9c below)     |

Once reachable, wire the alias so kit imports resolve:

```ts
// vite.config.ts
resolve: { alias: { '@rascal-ui-kit': path.resolve(__dirname, './src/rascal-ui-kit') } }
```

```json
// tsconfig.json
"paths": { "@rascal-ui-kit/*": ["./src/rascal-ui-kit/*"] }
```

### 9b. Vercel-specific notes (no extra config beyond standard Vite)

- **Inter** — bundled via `@fontsource-variable/inter`, served from Vercel Edge. Aman.
- **Aptos** — never host. Resolves locally on Windows 11 / M365 users; everyone else falls through to Inter. This is by design.
- **Material Symbols** — pulled from Google Fonts CDN via `<link>` in `index.html`. Vercel is transparent to external CDNs.
- **Logo SVG** — drop in `public/` for static caching at Vercel Edge (preferred), or `src/assets/` for content-hashed bundling. SVG is tiny — `public/` is the simpler default.
- **SPA fallback** — add `vercel.json` rewrite to `/index.html` if using client-side routing.
- **Email templates** — font stack switches to `'Segoe UI', system-ui, -apple-system, sans-serif` (email clients strip web fonts). PNG logo, not SVG.

### 9c. Custom shadcn registry (for scale)

If 3+ apps will consume the kit, publish it as a `shadcn`-compatible registry deployed on Vercel:

1. Build `$KIT/registry.json` listing each component (`StatCard`, `SmartGateCard`, `LoadingSpinner`, …) with code + Tailwind tokens + dependencies.
2. Deploy kit folder as static site on Vercel (e.g. `rascal-ui-kit.vercel.app`).
3. Apps install via:
   ```bash
   npx shadcn@latest add https://rascal-ui-kit.vercel.app/r/stat-card.json
   ```
4. v0.dev can read custom registries directly — prompt it with the registry URL and it pulls kit components.

Reference: https://ui.shadcn.com/docs/registry (the spec is provider-neutral; Vercel just hosts it).

### 9d. Prompt template for AI design tools (low-technical, paste verbatim)

When generating UI with v0 / Lovable / Cursor, **paste this block in front of every prompt**. It's plain English so non-developers can use it too — no Tailwind jargon required for the human, but the constraints translate to kit-correct output.

```
Use these design rules — they are not optional:

VIBE
- Calm, quiet interface. Lots of whitespace. Feels like a tool that
  respects the user's attention, not a marketing page.
- Only get loud (bold colors, big numbers, urgency) when the data
  actually demands it. Everything else stays muted.

COLORS
- Background: very light off-white (warm, not blue-gray).
- Text: dark warm gray, almost black. Not pure black.
- Borders: very faint warm gray hairlines, not boxy outlines.
- Regular buttons: near-black with white text.
- Submit / Confirm / Save buttons: forest green with white text.
- Brand navy blue: ONLY on the sidebar and the login screen.
  Never use brand navy for buttons inside the app.
- Warning: amber / soft orange.
- Error: muted red.
- Info: clean blue.

TYPE
- Font: clean modern sans-serif, similar to Microsoft's "Aptos" or
  open-source "Inter". Falls back to system fonts.
- Numbers in tables and stats: use tabular figures so digits line up.
- Headings are semibold and slightly tighter; body text is regular weight.

ICONS
- Use simple line icons (Material Symbols Rounded style — thin
  rounded strokes, no fills, no shadows).
- Emoji is allowed but ONLY these four: 👋 (greeting), ✓ (done),
  ⚠️ (warning), 🎯 (objective). No other emoji anywhere.
  Keep it minimal and clean — usually zero or one emoji per page.

LOADING
- Every loading state (app boot, login wait, page switch, blocker)
  must show the company logo in the center with a thin ring rotating
  around it. Never use a lone spinner without the logo. Never use
  just the word "Loading…" with no logo.

LAYOUT
- Pages are centered, max width around 1000–1200px.
- Cards: white background, gentle rounded corners, hairline border,
  no drop shadow (or very subtle if at all).
- Tables look like Notion — no card per row, just thin lines between
  rows and clean typography.

COPY
- Plain language, short sentences. Imperative verbs on buttons
  ("Open", "Submit", "Send"). No marketing words ("powerful",
  "delightful", "amazing"). No "Oops!" or "Whoops!".

TASK
<describe what you want here, in plain English>
```

After v0 / Lovable / Cursor produces code, run it back through this skill before merging:

```
/rascal-ui

Review this AI-generated component against the Rascal UI Kit
guardrails. Fix every token that drifted: gray/slate/zinc → stone,

[paste AI output]
```

The skill will run the 11-point sanity check and rewrite class strings to be kit-correct. AI tools rarely get the kit right first try — this lint step is part of the standard workflow, not a fallback.

### 9f. Anti-patterns specific to AI tooling

Common drift to watch for in AI-generated output:

1. `bg-gray-50` / `bg-zinc-50` page bg — **fix:** `bg-stone-50`.
2. `bg-blue-600` "submit" buttons — **fix:** `bg-emerald-700`.
3. Lucide / Heroicons-solid / Font Awesome icons — **fix:** Material Symbols Rounded.
4. New emoji on dashboards (🚀, ✨, 🔥, 💡, 📊) — **fix:** Material Symbol Rounded glyph or drop entirely. The minimal allowlist (👋 ✓ ⚠️ 🎯) is intentional.
5. Drop-shadow cards (`shadow-md` on every card) — **fix:** hairline `border-stone-200` instead.
6. Lone `<div className="animate-spin ...">` for loading — **fix:** branded `LoadingSpinner`.
7. Centered "Loading…" text with no logo — **fix:** branded `LoadingSpinner` with `logoSrc`.
8. Skeleton-only loader for auth boundary or app boot — **fix:** branded `LoadingSpinner` (skeletons are fine *inside* an authenticated page once the shell has loaded).
